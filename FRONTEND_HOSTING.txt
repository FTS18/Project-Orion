FRONTEND HOSTING GUIDE - Project Orion
=====================================

Your frontend is fully deployable separately! Here's how:

## QUICK START

### Local Build
npm run build:client:prod
# Creates: dist/client/ (production-ready build)

### Deploy Directory
Upload the entire "dist/client" folder to any static hosting

---

## HOSTING OPTIONS

### 1. VERCEL (Recommended - Zero Config)
- No build config needed (auto-detects)
- Free tier available
- Global CDN by default
- Steps:
  1. Push to GitHub
  2. Go to vercel.com, connect repo
  3. Set VITE_API_URL environment variable
  4. Deploy! ✅

### 2. NETLIFY
- Drag & drop or Git integration
- Free tier available
- netlify.toml already configured
- Steps:
  1. Push to GitHub
  2. Go to netlify.com, connect repo
  3. Set VITE_API_URL environment variable
  4. Deploy! ✅

### 3. AWS AMPLIFY
- Full AWS integration
- Free tier available
- Steps:
  1. Push to GitHub
  2. AWS Amplify → New app
  3. Connect repo, select "main" branch
  4. Set VITE_API_URL in environment variables
  5. Deploy! ✅

### 4. CLOUDFLARE PAGES
- Ultra-fast global CDN
- Free tier available
- Steps:
  1. Push to GitHub
  2. Cloudflare → Pages → Create project
  3. Connect repo
  4. Build command: npm run build:client:prod
  5. Output: dist/client
  6. Set VITE_API_URL
  7. Deploy! ✅

### 5. DOCKER + ANY CLOUD
- DigitalOcean, Heroku, AWS, Azure, GCP
- Use provided Dockerfile.frontend
- Commands:
  docker build -f Dockerfile.frontend -t project-orion-frontend .
  docker run -p 80:3000 project-orion-frontend

### 6. GITHUB PAGES (Free but limited)
- Works for static sites only
- No backend API support recommended
- Update vite.config.ts base: "/Project-Orion/"

## ENVIRONMENT CONFIGURATION

### Development (.env.local - current)
VITE_API_URL=http://localhost:5000

### Production (.env.production - created)
VITE_API_URL=https://your-backend-api.com

Replace "https://your-backend-api.com" with your actual backend URL

## API ROUTING

Frontend makes API calls to: VITE_API_URL + /api/*

Example requests:
- http://localhost:5000/api/customers
- http://localhost:5000/api/verify-kyc
- http://localhost:5000/api/underwrite

Ensure backend is accessible at this URL!

---

## CORS SETUP (Backend Required)

If hosting frontend on different domain, backend needs CORS:

Add to backend:
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://your-frontend-url.com"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
