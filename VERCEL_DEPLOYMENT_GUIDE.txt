## QUICK START FOR VERCEL

### Step 1: Push to GitHub
git add -A
git commit -m "Add mock data for showcase"
git push origin main

### Step 2: Deploy to Vercel
1. Go to vercel.com
2. Click "Add New Project"
3. Select your GitHub repository
4. Click "Import"
5. Wait for build to complete
6. Your site is now live! ✅

### Step 3 (Optional): Connect Backend
To use real backend instead of demo data:

1. On Vercel Dashboard
2. Go to Settings → Environment Variables
3. Add: VITE_API_URL = https://your-backend-api.com
4. Redeploy
5. Demo banner disappears, real data shown

## CONNECTING REAL BACKEND

When ready to use real backend:

### 1. Deploy Backend
- Use Railway, Render, Heroku, or AWS
- Note the backend URL (e.g., https://api.example.com)

### 2. Update Vercel Environment
- Go to Vercel project settings
- Add: VITE_API_URL = https://api.example.com
- Redeploy

### 3. Configure Backend CORS
Backend must allow requests from Vercel domain:

```python
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "https://your-vercel-domain.vercel.app",
        "http://localhost:3000",  # local dev
    ],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

## TROUBLESHOOTING

### Page loads but shows loading spinner forever
→ Backend not connected and demo mode not activated
→ Solution: Refresh page, or set VITE_API_URL empty

### Getting timeout errors
→ Backend is not accessible or slow
→ Solution: 
   a) Deploy backend to fast server
   b) Or remove VITE_API_URL to use demo mode

### Mock data not showing
→ setupMockFetch() not called
→ Solution: Rebuild and redeploy

### Dark mode not working
→ Theme preference not loading
→ Solution: Hard refresh browser (Ctrl+Shift+R)
